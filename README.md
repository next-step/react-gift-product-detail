#Step3 테스트 코드 작성하기

1. 멘토님과 파일 구조가 달라, 제 파일에서 재사용되는 폼과 그 폼에 대한 타이포그래피에 대한 테스트를 진행했습니다.
   1. src/components/SenderForm/styles.ts의 Input - 발신자 폼과 수신자 모달에서 공통으로 사용
   2. src/pages/LoginPage/styles.ts의 Input - 로그인 페이지의 입력 필드
      이 두가지에 대한 테스트를 진행했습니다.

2. 로그인 페이지에 대한 구현한 시나리오는 다음과 같습니다.
   처음 로그인 페이지 화면 : 이메일폼, 비밀번호 폼, 버튼 전부다 disabled

   이메일 폼 입력 시나리오:
   1. 이메일 폼이 포커스 된 상태일 때, 특정 값을 쓰다가 다시 삭제해서 빈 값이 되면, ID를 입력해주세요 에러메세지 발생
   2. 이메일 폼이 포커스 된 상태일 때, 이메일 형식이 아닌 값을 입력했을 때, ID는 이메일 형식으로 입력해주세요. 에러 메세지 발생
   3. 이메일 폼이 포커스 된 상태일 때, 올바른 이메일 형식을 입력했을 때, 에러메세지가 사라져야 한다.

   로그인 폼 입력 시나리오:
   1. 비밀번호 폼이 포커스 된 상태일 때, 특정 값을 쓰다가 다시 삭제해서 빈 값이 되면, 비밀번호를 입력해주세요 에러메세지 발생
   2. 비밀번호 폼이 포커스 된 상태일 때, 비밀번호가 8자 미만일 때, 비밀번호는 8자 이상 입력해주세요 에러메세지가 표시되어야 한다.
   3. 비밀번호 폼이 포커스 된 상태일 때, 비밀번호가 8자 이상일 때, 에러메세지가 사라져야 한다.

   버튼 시나리오
   1. 로그인 페이지가 렌더링 되었을 때, 이메일과 비밀번호가 형식에 모두 맞을 때, 로그인 버튼이 enabled되어야한다.
   2. 로그인 페이지가 렌더링 되었을 때, 이메일이나 비밀번호가 형식에 맞지 않을 때, 로그인 버튼이 disabled 상태여야 한다.
   3. 로그인 페이지가 렌더링 되었을 때, 이메일과 비밀번호가 형식에 모두 맞을 때, 로그인 버튼을 클릭하면 로그인 성공 시 메인 페이지로 이동해야 한다.

   이러한 시나리오를 사전 설정하고 테스트도 이에 맞게 구현하였습니다.

3. MSW를 활용한 선물하기 랭킹섹션
   1. 필터 12개의 조합에 대해 각각 목데이터를 설정하고 (몇개 복붙했습니다) 각 버튼 클릭시 올바른 데이터가 들어오는지 , 데이터가 없을 시에 메세지가 잘 나오는지, 로컬스토리지에 저장은 잘되는지, 등 에 대한 테스트를 진행했습니다.
